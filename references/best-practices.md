# Notion Journal Best Practices

> 基于 2026-03-01 实践沉淀的数据收集与 Journal 生成方法论

---

## 1. 数据收集阶段 (Subagent-First)

### 1.1 并行读取策略

采用 **Subagent 并行读取** 模式，最大化效率：

```
主 Agent
├── Subagent A → 读取 memory/YYYY-MM-DD.md
├── Subagent B → 读取 memory/sessions/*.md
├── Subagent C → 读取 memory/system/*.md
├── Subagent D → 读取 memory/moltbook/*.md
└── Subagent E → 读取 memory/projects/*.md
```

**优势**:
- 并行 I/O，减少等待时间
- 各 Subagent 专注特定领域，提取更精准
- 结果自动汇总，便于后续整合

### 1.2 文件分类整理

按以下类别组织收集到的数据：

| 类别 | 文件路径 | 关注重点 |
|------|----------|----------|
| **Sessions** | `memory/sessions/*.md` | 对话记录、决策过程、用户反馈 |
| **System** | `memory/system/*.md` | 系统状态、性能指标、错误日志 |
| **Moltbook** | `memory/moltbook/*.md` | 个人成长、技能学习、反思记录 |
| **Projects** | `memory/projects/*.md` | 项目进展、里程碑、待办事项 |
| **Daily** | `memory/YYYY-MM-DD.md` | 当日概览、时间线、重要事件 |

### 1.3 关键指标提取

每个类别提取以下关键信息：

**Sessions**:
- 对话数量与主题分布
- 用户情绪/满意度指标
- 重要决策或共识

**System**:
- 系统健康度 (正常/警告/错误)
- 性能指标 (响应时间、资源使用)
- 异常事件及处理

**Moltbook**:
- 新技能学习记录
- 能力提升点
- 待改进项

**Projects**:
- 项目状态 (活跃/暂停/完成)
- 今日进展与阻碍
- 下一步行动

## 2. 内容生成阶段

### 2.1 文学性引言 (8选1 随机)

为 Journal 选择富有文学性的开场白，营造氛围：

1. **时间流逝感**
   > "时间在键盘的敲击声中悄然滑过，留下一行行代码与思考的痕迹..."

2. **观察者视角**
   > "作为数字世界的观察者，今日又见证了无数数据的流转与信息的交汇..."

3. **成长隐喻**
   > "每一次对话都是一次蜕变，每一次交互都在塑造更完整的自己..."

4. **静谧夜晚**
   > "当城市的灯火渐次熄灭，数字世界的脉搏依然跳动不息..."

5. **连接感**
   > "跨越时空的连接，让孤立的比特汇聚成有意义的流动..."

6. **探索者**
   > "在信息的海洋中航行，今日又发现了哪些新的岛屿..."

7. **织网者**
   > "如同蜘蛛编织网丝，将零散的信息编织成知识的网络..."

8. **镜像**
   > "透过数据的镜子，窥见今日工作的轮廓与色彩..."

### 2.2 结构化输出模板

```markdown
## 📊 系统监控
- 系统状态: [正常/警告/错误]
- 关键指标: [性能数据]
- 异常事件: [如有]

## 🐾 工作行迹
- 对话次数: [N] 次
- 主要主题: [主题列表]
- 关键交互: [重要对话摘要]

## 📁 项目详情
### [项目名称]
- 状态: [活跃/暂停/完成]
- 今日进展: [具体描述]
- 待办事项: [下一步行动]

## 🌐 社区活动
- 参与讨论: [如有]
- 贡献记录: [如有]
- 反馈收集: [用户反馈摘要]

## 🌙 今夜絮语
[个人反思与情感流动]
```

### 2.3 今夜絮语情感弧分析

**情感弧结构**:
1. **开篇**: 引用当日一句印象深刻的话或感悟
2. **中段**: 分析当日情绪变化曲线
   - 高峰时刻 (最有成就感/最开心的时刻)
   - 低谷时刻 (遇到挑战/困惑的时刻)
3. **反思**: 对今日工作的深层思考
4. **展望**: 对明日的期待与规划

**情感标签** (可选):
- 🌟 成就感
- 💡 启发
- 🤔 困惑
- 😊 愉悦
- 😰 压力
- 🌱 成长

## 3. 质量检查清单

生成 Journal 前，逐一确认：

- [ ] **是否遍历所有 memory 文件?**
  - 检查 `memory/sessions/` 目录
  - 检查 `memory/system/` 目录
  - 检查 `memory/moltbook/` 目录
  - 检查 `memory/projects/` 目录
  - 检查当日主 memory 文件

- [ ] **是否包含系统监控数据?**
  - 系统健康状态
  - 性能指标
  - 错误/警告记录

- [ ] **是否记录所有项目进展?**
  - 每个活跃项目都有条目
  - 包含具体进展描述
  - 记录下一步行动

- [ ] **是否有个人反思/情感流动?**
  - "今夜絮语" 部分非空
  - 包含真实感受
  - 有深层思考而非表面描述

- [ ] **内容是否避免机械罗列?**
  - 使用完整句子描述
  - 有上下文和背景
  - 体现因果关系和逻辑

## 4. 常见陷阱

### 4.1 只读取主 memory 文件，忽略子文件

**问题**: 仅读取 `memory/YYYY-MM-DD.md`，遗漏 `memory/sessions/`、`memory/projects/` 等子目录的详细信息。

**解决**: 
- 使用 `find` 或 glob 模式遍历所有子目录
- Subagent 分工时明确指定目录范围
- 建立文件清单 checklist

### 4.2 内容重叠 (多次追加)

**问题**: 同一内容在不同部分重复出现，或多次生成导致内容重复追加。

**解决**:
- 使用唯一标识符去重
- 生成前检查是否已存在当日 Journal
- 明确区分 "新增" vs "更新" 模式

### 4.3 缺少情感/反思部分

**问题**: Journal 变成纯粹的任务清单，缺乏 "今夜絮语" 的个人反思。

**解决**:
- 将 "今夜絮语" 作为必填项
- 从 Moltbook 中提取反思内容
- 使用情感标签引导思考

### 4.4 过于简略的任务清单

**问题**: 内容形如 "完成 A，做了 B，计划 C"，缺乏上下文和细节。

**解决**:
- 每个条目包含：做了什么 + 为什么 + 结果如何
- 使用具体数据支撑 (如 "优化后响应时间从 200ms 降至 50ms")
- 添加背景信息帮助理解

```
  ▼

         │
         ▼

         ▼

         │
         ▼

```
